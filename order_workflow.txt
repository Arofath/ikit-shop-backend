🔄 លំហូរនៃការបញ្ជាទិញ (The Standard Flow)
១. ដំណាក់កាលប្រព័ន្ធស្វ័យប្រវត្តិឆែក (System Auto-Check):
នៅពេលអតិថិជនចុចប៊ូតុង "Place Order" (បញ្ជាក់ការទិញ) ភ្លាម កូដ Backend Laravel របស់លោកអ្នកត្រូវធ្វើការងារទាំងនេះក្នុងប្រភាគវិនាទី៖

ឆែកមើលតើទំនិញក្នុងកន្ត្រកនៅមានស្តុកឬទេ?

តើព័ត៌មានដែលគាត់វាយបញ្ចូល (ឈ្មោះ, លេខទូរស័ព្ទ, អាសយដ្ឋាន) គ្រប់គ្រាន់ហើយឬនៅ?

បើត្រឹមត្រូវអស់ហើយ ប្រព័ន្ធនឹង Save ទិន្នន័យចូល Database ភ្លាមៗ ដោយកំណត់ស្ថានភាពវិក្កយបត្រ (Order Status) ជាបឋមគឺ pending (រង់ចាំការបញ្ជាក់)។

បន្ទាប់មកកាត់ស្តុកទំនិញចេញពីឃ្លាំង, លុបទំនិញពីកន្ត្រកគាត់ចេញ និងរុញគាត់ទៅទំព័រ "ទិញជោគជ័យ (Thank You Page)"។

២. ដំណាក់កាលជូនដំណឹង (Notification to Admin):
ភ្លាមៗបន្ទាប់ពី Save ចូល Database ជោគជ័យ វេបសាយនឹងផ្ញើសារជូនដំណឹងទៅកាន់ Admin ស្វ័យប្រវត្តិ។ (នៅស្រុកខ្មែរ គេនិយមភ្ជាប់កូដជាមួយ Telegram Bot។ ពេលមានគេទិញភ្លាម Bot នឹងលោតសារប្រាប់ Admin ថា៖ "🎉 មានការបញ្ជាទិញថ្មី! វិក្កយបត្រ #001, តម្លៃ: $150, លេខទូរស័ព្ទ: 012xxx...")។

៣. ដំណាក់កាល Admin ធ្វើការ (Admin Manual Check):
នៅពេល Admin ឃើញសារជូនដំណឹង Admin នឹងចូលទៅកាន់ផ្ទាំងគ្រប់គ្រង (Admin Panel) ដើម្បី៖

មើលព័ត៌មានលម្អិត។

តេកាត់ (Call) ទៅកាន់អតិថិជនដើម្បីបញ្ជាក់ការទិញ និងទីតាំង។

ឆែកមើលវិក្កយបត្របង់លុយ (បើគាត់រើស Bank Transfer)។

បន្ទាប់មកទើប Admin ជាអ្នកចុចប្តូរស្ថានភាព (Status) ពី pending ទៅជាស្ថានភាពបន្ទាប់។

🚦 ការកំណត់ Order Status (Status Management Idea)
ដើម្បីឱ្យប្រព័ន្ធរបស់លោកអ្នកច្បាស់លាស់ លោកអ្នកគួរកំណត់ Status របស់ Order ជាជំហានៗដូចខាងក្រោម៖

pending (រង់ចាំការបញ្ជាក់/រង់ចាំបង់ប្រាក់): ជា Status ដំបូងគេពេលអតិថិជនទើបតែចុចទិញរួច។ ប្រសិនបើគាត់រើសយកការបង់ប្រាក់តាមធនាគារ (Bank Transfer) គាត់នឹងស្ថិតក្នុង Status នេះរហូតដល់គាត់ផ្ញើវិក្កយបត្រមក។

processing (កំពុងរៀបចំទំនិញ): នៅពេល Admin តេទាក់ទងភ្ញៀវត្រូវហើយ ឬ ពិនិត្យឃើញថាគាត់បានវេរលុយត្រឹមត្រូវហើយ Admin នឹងប្តូរ Status មក processing ដើម្បីឱ្យក្រុមការងាររៀបចំឥវ៉ាន់ខ្ចប់។

shipped (កំពុងដឹកជញ្ជូន): នៅពេលប្រគល់ឥវ៉ាន់ទៅឱ្យអ្នកដឹក (Delivery Guy) Admin ប្តូរទៅ shipped។ ប្រព័ន្ធអាចនឹងផ្ញើសារប្រាប់អតិថិជនថាឥវ៉ាន់កំពុងធ្វើដំណើរទៅហើយ។

delivered / completed (ជោគជ័យ): ពេលអតិថិជនទទួលបានឥវ៉ាន់ និងបង់លុយរួចរាល់ (សម្រាប់ COD)។

cancelled (លុបចោល): ក្នុងករណីតេទៅភ្ញៀវអត់ចូល ភ្ញៀវបដិសេធមិនយក ឬ ទំនិញខូចខាត។ (ចំណាំ៖ ពេលប្តូរទៅ cancelled ត្រូវសរសេរកូដយកចំនួនស្តុក បូកបញ្ចូលទៅក្នុងឃ្លាំងវិញ)។

💳 គំនិតបន្ថែមលើ "ជម្រើសបង់ប្រាក់"
បើអតិថិជនរើស Cash on Delivery (COD): ពេលចុច Place Order វាចប់ត្រឹមហ្នឹង លោតទៅ Thank You Page តែម្តង។ លុយចាំទារពេលឥវ៉ាន់ទៅដល់។

បើអតិថិជនរើស Bank Transfer (ABA, ACLEDA...): ពេលគាត់រើសជម្រើសនេះ លោកអ្នកគួរតែលោតចេញ QR Code របស់ហាងឱ្យគាត់ស្កេន។ ហើយមានប៊ូតុង ឬកន្លែងសម្រាប់ឱ្យគាត់ Upload រូបភាពវិក្កយបត្រ (Screenshot) បញ្ជាក់ថាបានបង់លុយរួច។ រូបភាពនេះនឹងត្រូវ Save ចូល Database ភ្ជាប់ជាមួយ Order ដើម្បីឱ្យ Admin ងាយស្រួលឆែក។
